Cara Kerja Internet

Internet adalah jaringan global yang memungkinkan komputer dan perangkat lain untuk bertukar data. Data dikirim dalam bentuk paket melalui jaringan yang saling terhubung, termasuk kabel fiber optik, satelit, atau nirkabel. Setiap perangkat yang terhubung ke internet memiliki alamat unik yang disebut alamat IP.

Internet bekerja dengan menggunakan protokol, atau aturan komunikasi, yang memungkinkan perangkat di seluruh dunia untuk saling berkomunikasi. Protokol utama yang digunakan untuk pertukaran data adalah TCP/IP (Transmission Control Protocol/Internet Protocol). TCP bertanggung jawab untuk memecah data menjadi paket-paket kecil yang dapat ditransmisikan melalui jaringan, sementara IP bertanggung jawab untuk mengarahkan paket-paket ini ke tujuan yang benar.

Selain kabel fisik dan nirkabel, infrastruktur internet juga melibatkan router, server, dan switch yang membantu mengarahkan lalu lintas data ke jalur yang paling efisien. Pengguna biasanya mengakses internet melalui ISP (Internet Service Provider), yang menghubungkan rumah dan kantor ke jaringan global ini.

HTTP & HTTPS

HTTP (Hypertext Transfer Protocol) adalah protokol yang memungkinkan pertukaran data antara klien (seperti browser) dan server. Ini memungkinkan kita untuk mengakses halaman web dengan mengetikkan URL di browser. HTTP digunakan untuk mengirim dan menerima informasi seperti dokumen, gambar, dan file lain melalui jaringan internet.

Namun, HTTP sendiri tidak menawarkan keamanan, yang berarti data yang dikirimkan dapat dilihat oleh pihak ketiga. HTTPS (Hypertext Transfer Protocol Secure) adalah versi aman dari HTTP, yang menggunakan SSL/TLS untuk mengenkripsi data yang dikirimkan antara klien dan server. Hal ini memastikan bahwa informasi pribadi, seperti data login atau informasi pembayaran, aman dari pencurian informasi oleh pihak ketiga.

SSL (Secure Sockets Layer) dan TLS (Transport Layer Security) adalah protokol enkripsi yang menciptakan koneksi aman. Saat Anda melihat simbol gembok di sebelah URL di browser, ini menandakan bahwa halaman tersebut menggunakan HTTPS, memberikan lapisan keamanan tambahan untuk pengguna.

Nama Domain

Nama domain adalah alamat unik yang digunakan untuk mengidentifikasi situs web di internet, seperti google.com. Alih-alih menggunakan alamat IP, kita menggunakan nama domain yang lebih mudah diingat. Nama domain ini dipetakan ke alamat IP melalui DNS (Domain Name System).

Domain terdiri dari beberapa bagian yang dipisahkan oleh titik, seperti "www.example.com". Bagian paling kanan, seperti ".com", disebut TLD (Top-Level Domain), dan berfungsi untuk mengidentifikasi kategori domain atau kode negara. Bagian "example" adalah nama domain yang dipilih oleh pemilik situs. Dalam hierarki DNS, domain juga bisa memiliki subdomain seperti "blog.example.com" untuk mengarahkan ke bagian tertentu dari situs tersebut.

Nama domain perlu didaftarkan melalui registrar domain, seperti GoDaddy atau Namecheap, dan memiliki durasi sewa yang perlu diperpanjang secara periodik. Proses ini memastikan bahwa pemilik domain memegang kendali atas alamat tersebut selama periode waktu tertentu.

Referensi: Cloudflare

Hosting

Hosting adalah layanan penyimpanan file situs web di server yang selalu terhubung ke internet. Penyedia hosting menjaga agar situs web dapat diakses setiap saat dari mana saja di seluruh dunia. Server hosting dapat berupa server fisik yang kuat atau server virtual yang di-host di cloud.

Jenis layanan hosting bervariasi, termasuk shared hosting (satu server digunakan oleh beberapa situs web), VPS (Virtual Private Server), dan dedicated server (server khusus untuk satu situs web). Shared hosting cocok untuk situs yang sederhana dan memiliki lalu lintas rendah, sementara dedicated server digunakan untuk situs dengan lalu lintas tinggi dan memerlukan kontrol penuh atas server.

Hosting juga menyediakan fitur seperti penyimpanan data, email, dan database. Ketika seseorang mengakses situs web Anda, permintaan dikirim ke server hosting, yang kemudian mengirimkan kembali data halaman yang diminta untuk ditampilkan di browser pengguna.

Referensi: Hostgator

DNS

DNS (Domain Name System) adalah sistem yang menerjemahkan nama domain menjadi alamat IP. Saat kita mengetik nama domain di browser, DNS lookup dilakukan untuk menemukan alamat IP dari server yang dituju, sehingga pengguna dapat terhubung ke situs web tersebut.

DNS dapat dianggap sebagai buku telepon internet, di mana setiap nama domain dipetakan ke alamat IP server. Proses penerjemahan ini dilakukan oleh server DNS yang tersebar di seluruh dunia. Jika server DNS tidak memiliki catatan untuk domain tertentu, permintaan akan diteruskan ke server lain hingga alamat IP yang benar ditemukan.

DNS juga menyediakan redundansi untuk memastikan keandalan. Ketika satu server DNS tidak dapat merespons permintaan, server lain dapat mengambil alih, memastikan bahwa pengguna tidak menghadapi gangguan dalam mengakses situs yang diinginkan.

Referensi: Cloudflare

Browser

Browser adalah perangkat lunak yang digunakan untuk menampilkan informasi dari internet. Browser mengirimkan permintaan HTTP/HTTPS ke server, menerima data, dan kemudian menampilkannya kepada pengguna. Contoh browser termasuk Google Chrome, Mozilla Firefox, dan Safari.

Browser tidak hanya menampilkan halaman web, tetapi juga menafsirkan kode HTML, CSS, dan JavaScript untuk memformat halaman, menambah gaya, dan menjalankan skrip. Selain itu, browser modern memiliki berbagai fitur untuk meningkatkan keamanan, seperti mode private browsing, perlindungan terhadap phising, dan pemblokiran iklan yang mencurigakan.

Setiap browser memiliki rendering engine yang bertanggung jawab untuk mengubah kode menjadi elemen visual di layar. Contohnya, Google Chrome menggunakan Blink sebagai rendering engine. Selain itu, browser juga menggunakan cache untuk menyimpan data halaman web yang sering dikunjungi, yang membantu mempercepat waktu muat halaman.
